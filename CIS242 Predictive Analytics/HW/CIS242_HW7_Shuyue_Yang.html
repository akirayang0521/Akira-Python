6. Is that the right number of clusters? How do you know? Describe the quality of the clusters - be specific.¶

In [ ]:
# code or markdown here

We want our clusters to be "unique." In another word, we want the sum of squares within each cluster to be small because it means the cluster is cohesive.
We also want each cluster to be different from its neighboring clusters.
By comparing the Within Sum of Squares (withinss) and Between Sum of Squares (betweenss) of cluster models with 2, 3, and 4 clusters,
we can see that k=4 has the smallest withinss and largest betweenss.

Also, by comparing the cohesion and separation of cluster models with 2, 3, and 4 clusters,
we can see that k=4 has the most separation and the most cluster cohesion,
which means 4 can be the most appropriate number of clusters.
(Cohesion is measured by minimizing the ratio of withinss/totalss.
Separation is measured by maximizing the ratio of betweenss/totalss.)

When k=4 (refer to the above graph), we can see that the only outlier has been separated into a solitary cluster.
The borders of each cluster are relatively clear,
except for that the yellow color cluster somehow lies across the purple and cyan ones.

import numpy as np

def getBetweenss(data, kmeans_clusters):
    betweenss = 0
    mean = data.mean()
    for i in np.arange(len(data)):
        betweenss += np.sum((kmeans_clusters.cluster_centers_[kmeans_clusters.labels_[i]]-mean)**2)
    return betweenss
